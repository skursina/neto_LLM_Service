# LLM Сервис

LLM Сервис на FastAPI для суммаризации текста с использованием больших языковых моделей (LLM).

## Обзор

Этот сервис предоставляет простое HTTP API для генерации кратких содержаний больших текстовых фрагментов. Он разработан с четким разделением ответственности, надежной обработкой ошибок и функциями для продакшена, такими как кэширование и механизмы повторных попыток.

## Возможности

*   **HTTP API**: RESTful интерфейс на основе FastAPI.
*   **Интеграция с LLM**: Гибкий клиент для взаимодействия с различными провайдерами LLM через API, совместимое с OpenAI.
*   **Отказоустойчивость**: Реализует логику повторных попыток для временных сетевых ошибок и фолбэк-суммаризатор при недоступности LLM.
*   **Кэширование**: Использует кэш в памяти с TTL для улучшения производительности при повторяющихся запросах.
*   **Структурированное логирование**: Комплексное логирование с разными уровнями серьезности для мониторинга и отладки.
*   **Управление конфигурацией**: Все настройки управляются через переменные окружения.
*   **Валидация данных**: Модели Pydantic обеспечивают целостность входных данных.

## Архитектура

Сервис следует многоуровневой архитектуре:

1.  **API-слой (`api/`)**: Обрабатывает HTTP-запросы и ответы. Определяет POST-эндпоинт `/summarize` и GET-эндпоинт `/health`.
2.  **Слой бизнес-логики (`services/`)**: Координирует процесс суммаризации. `summarizer_service` проверяет кэш, вызывает LLM, обрабатывает ошибки и применяет фолбэк.
3.  **Слой LLM (`llm/`)**: Управляет взаимодействием со внешней LLM. Модуль `client` обрабатывает вызовы API с повторными попытками, таймаутами и трансляцией ошибок.
4.  **Конфигурация и общее (`config/`, `models/`)**: Содержит настройки приложения, модели Pydantic для запросов/ответов и общие утилиты.

## Предварительные требования

*   Python 3.9 или выше
*   API-ключ для провайдера LLM, совместимого с OpenAI (например, от api.llm7.io)

## Установка

1.  Склонируйте репозиторий.
2.  Создайте виртуальное окружение:
    ```bash
    python -m venv .venv
    source .venv/bin/activate # В Windows: `.venv\Scripts\activate`
    ```
3.  Установите зависимости:
    ```bash
    pip install -r requirements.txt
    ```

## Конфигурация

1.  Скопируйте пример файла окружения:
    ```bash
    cp .env.example .env
    ```
2.  Отредактируйте файл `.env` и укажите ваш API-ключ и другие настройки:
    ```env
    OPENAI_API_KEY=ваш_реальный_api_ключ_здесь
    LLM_BASE_URL=https://api.llm7.io/v1
    MODEL_NAME=gpt-4o-mini
    # При необходимости скорректируйте другие настройки
    ```

> **Важно**: Никогда не коммитьте файл `.env` в систему контроля версий. Он уже включен в `.gitignore`.

## Использование

1.  Запустите сервер разработки:
    ```bash
    uvicorn main:app --reload
    ```
2.  API будет доступно по адресу `http://localhost:8000`.

### API Эндпоинты

*   **`GET /health`**: Проверка работоспособности. Возвращает `{"status": "ok"}`, если сервис работает.
*   **`POST /summarize`**: Сгенерировать краткое содержание.

    **Тело запроса (JSON):**
    ```json
    {
      "text": "Ваш большой текст для суммаризации..."
    }
    ```

    **Ответ (JSON):**
    ```json
    {
      "summary": "Краткое содержание предоставленного текста."
    }
    ```

```bash
curl -X POST "http://localhost:8000/summarize" \
  -H "Content-Type: application/json" \
```

## Тестирование

Модульные тесты находятся в директории `tests/`. Запустите их с помощью:

```bash
pytest
```

## Линтинг

В проекте используется `ruff` для линтинга. Запустите его с помощью:

```bash
ruff check .
```
