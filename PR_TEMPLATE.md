# Pull Request: Реализация LLM-сервиса суммаризации

## Описание изменений

В данном PR реализован production-ready сервис на FastAPI для суммаризации текста с использованием больших языковых моделей (LLM). Сервис разработан с четким разделением ответственности, обеспечивает отказоустойчивость и включает в себя все необходимые компоненты для промышленного использования.

## Архитектура и основные компоненты

Сервис следует модульной многоуровневой архитектуре:

*   **API-слой (`api/`)**: Реализует HTTP-интерфейс с помощью FastAPI. Содержит эндпоинт `/summarize` для обработки запросов и `/health` для проверки работоспособности.
*   **Слой бизнес-логики (`services/`)**: Координирует процесс обработки. `summarizer_service` отвечает за проверку кеша, вызов LLM и применение fallback-логики.
*   **Слой LLM (`llm/`)**: Управляет взаимодействием с внешними провайдерами LLM. Модуль `client` обеспечивает надежные вызовы с повторными попытками и таймаутами.
*   **Конфигурация и общее (`config/`, `models/`)**: Содержит настройки приложения (через переменные окружения), модели данных Pydantic и общие утилиты.
